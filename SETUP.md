## ЧАСТЬ 1 — FIREBASE (база данных + авторизация)

### Шаг 1: Создать аккаунт Google

Если уже есть — пропускай.
Перейди на **gmail.com**, нажми «Создать аккаунт».

---

### Шаг 2: Создать проект Firebase

1. Перейди на **https://firebase.google.com**
2. Нажми синюю кнопку **«Начать»** или **«Get started»**
3. Войди через свой Google аккаунт
4. Нажми **«Создать проект»** (большой плюс)
5. Введи имя проекта: например `chaos-control-app`
6. **Google Analytics** — можно отключить (переключатель влево), нажми **«Создать проект»**
7. Подожди ~30 секунд, нажми **«Продолжить»**

---

### Шаг 3: Включить авторизацию Google

1. В левом меню Firebase нажми **«Build»** → **«Authentication»**
2. Нажми **«Get started»** / **«Начать»**
3. Перейди на вкладку **«Sign-in method»** / **«Способ входа»**
4. Найди **Google**, нажми на него
5. Переключатель **«Enable»** → включи (синий)
6. В поле **«Project support email»** выбери свою почту
7. Нажми **«Save»** / **«Сохранить»**

#### Авторизация Яндекс (необязательно, чуть сложнее):

> Если хочешь только Google — пропусти этот блок.

1. В том же разделе Authentication → Sign-in method найди **«Add new provider»**
2. Выбери **«SAML»** или **«OpenID Connect»** (Яндекса нет в стандартном списке — нужен **«OpenID Connect»**)
3. Перейди на **https://oauth.yandex.ru**
4. Создай приложение, укажи redirect URI: `https://ВАШ-ПРОЕКТ.firebaseapp.com/__/auth/handler`
5. Получи **Client ID** и **Client secret** из Яндекс OAuth
6. В Firebase: Name → `yandex.com`, Client ID и Client Secret — вставь из Яндекс
7. Issuer: `https://login.yandex.ru`
8. Сохрани

---

### Шаг 4: Создать базу данных Firestore

1. В левом меню Firebase → **«Build»** → **«Firestore Database»**
2. Нажми **«Create database»** / **«Создать базу данных»**
3. Выбери режим **«Start in test mode»** / **«Режим тестирования»** (для разработки)
4. Выбери регион: **`europe-west3` (Frankfurt)** — ближайший к России
5. Нажми **«Enable»** / **«Включить»**

> ⚠️ **Режим тестирования** позволяет читать/писать всем. Для личного использования — окей. Позже можно настроить правила безопасности (см. Часть 4 этой инструкции).

---

### Шаг 5: Зарегистрировать веб-приложение и получить конфиг

1. В Firebase Console — нажми на шестерёнку ⚙️ рядом с «Project Overview» → **«Project settings»**
2. Пролистай вниз до раздела **«Your apps»**
3. Нажми иконку **`</>`** (веб-приложение)
4. В поле **«App nickname»** введи: `chaos-control-web`
5. Галочку «Firebase Hosting» **НЕ ставь** — будем использовать GitHub Pages
6. Нажми **«Register app»** / **«Зарегистрировать»**
7. Ты увидишь блок кода — **скопируй значения** из него:

```javascript
const firebaseConfig = {
  apiKey: "AIzaSy...",            // ← скопируй это
  authDomain: "мой-проект.firebaseapp.com",
  projectId: "мой-проект",
  storageBucket: "мой-проект.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abc123"
};
```

8. Нажми **«Continue to console»**

---

### Шаг 6: Вставить конфиг в файл приложения

1. Открой файл `js/firebase-config.js` в любом текстовом редакторе
   (Блокнот на Windows, TextEdit на Mac, или VSCode)
2. Найди секцию `const firebaseConfig = {` 
3. Замени все `"ВСТАВЬТЕ_СЮДА"` и `"ВАШ-ПРОЕКТ"` на **реальные значения** из шага 5
4. Сохрани файл

**Пример до:**
```javascript
apiKey: "ВСТАВЬТЕ_СЮДА",
projectId: "ВАШ-ПРОЕКТ",
```

**Пример после:**
```javascript
apiKey: "AIzaSyBxxxxxxxxxxxxxxxxxxxxxxx",
projectId: "chaos-control-app",
```

---

### Шаг 7: Добавить домен GitHub Pages в Firebase

Без этого авторизация Google работать не будет!

1. В Firebase → **Authentication** → **Settings** → **Authorized domains**
2. Нажми **«Add domain»**
3. Введи: `ТВО_ЛОГИН_GITHUB.github.io`
   (например: `ivanov.github.io`)
4. Сохрани

### Шаг 10: Загрузить файлы — 2 способа

#### Способ А: Через браузер (проще, без установки программ)

1. На странице репозитория нажми **«uploading an existing file»** или **«Add file»** → **«Upload files»**
2. Перетащи в окно браузера **все файлы проекта** сохраняя структуру папок:
   ```
   index.html
   css/
     main.css
   js/
     firebase-config.js
     auth.js
     db.js
     app.js
     modules/
       today.js
       projects.js
       chaos.js
   ```
   > ⚠️ GitHub не принимает папки через drag-and-drop напрямую — нужен способ Б или специальная техника: загружать файлы из каждой папки отдельно через кнопку «choose your files».

#### Способ Б: Через GitHub Desktop (рекомендую, проще для папок)

1. Скачай **GitHub Desktop**: https://desktop.github.com
2. Установи и войди в свой GitHub аккаунт
3. Нажми **«Clone a repository»** → выбери `chaos-control`
4. Укажи папку на компьютере куда сохранить
5. Скопируй все файлы проекта в эту папку (заменяя если есть)
6. В GitHub Desktop появится список изменений — напиши в поле «Summary»: `Initial commit`
7. Нажми **«Commit to main»**
8. Нажми **«Push origin»** (синяя кнопка)

---

### Шаг 11: Включить GitHub Pages

1. На странице репозитория → **Settings** (вкладка)
2. В левом меню → **«Pages»**
3. В разделе **«Branch»** выбери: **`main`** из выпадающего списка
4. Папку оставь **`/ (root)`**
5. Нажми **«Save»**
6. Подожди **1-3 минуты**
7. Обнови страницу — появится зелёный баннер с адресом:
   `Your site is live at https://ТВОЙ_ЛОГИН.github.io/chaos-control`

## ЧАСТЬ 3 — ПРОВЕРКА

### Шаг 12: Открыть приложение

1. Перейди по адресу: `https://ТВОЙ_ЛОГИН.github.io/chaos-control`
2. Должен появиться экран входа с кнопкой «Войти через Google»

4. Выбери свой аккаунт

#### Если что-то не работает:
- **Белый экран** → открой консоль браузера (F12 → Console) и посмотри ошибки
- **«auth/unauthorized-domain»** → ты не добавил домен в Firebase (Шаг 7)
- **«Firebase: Error (auth/configuration-not-found)»** → ошибка в конфиге (Шаг 6)
- **«Cannot read properties of undefined»** → файлы загружены не полностью

## ЧАСТЬ 4 — БЕЗОПАСНОСТЬ (важно!)

После того как приложение заработало, нужно ограничить доступ к базе данных.
Сейчас в «тестовом режиме» базу могут читать все.

### Настройка правил Firestore:

1. Firebase Console → **Firestore** → **Rules**
2. Замени содержимое на:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Каждый пользователь может читать и писать только СВОИ данные
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

3. Нажми **«Publish»**

Теперь данные каждого пользователя изолированы.

## ЧАСТЬ 6 — СТРУКТУРА ФАЙЛОВ

```
chaos-control/
│
├── index.html              ← Главная страница (вся разметка)
│
├── css/
│   └── main.css            ← Все стили (цвета, шрифты, размеры)
│
└── js/
    ├── firebase-config.js  ← ⚠️ ТВОИ КЛЮЧИ FIREBASE (не публикуй!)
    ├── auth.js             ← Авторизация Google/Яндекс
    ├── db.js               ← Работа с базой данных
    ├── app.js              ← Главный файл (навигация, координация)
    │
    └── modules/            ← Модули (каждый = один раздел приложения)
        ├── today.js        ← План дня
        ├── projects.js     ← Проекты и категории
        └── chaos.js        ← Место Хаоса
```

### Как добавить новый модуль (в будущем):
1. Создай файл `js/modules/НовыйМодуль.js`
2. Добавь в `index.html` строку: `<script type="module" src="js/modules/НовыйМодуль.js"></script>`
3. Добавь кнопку в нижнюю навигацию в `index.html`
4. Добавь в `app.js` вызов инициализации модуля

## ЧАСТЬ 7 — FAQ

**В: Приложение работает только онлайн?**
О: Да, требует интернет для синхронизации. Оффлайн-режим можно добавить позже через Service Worker.

**В: Данные в безопасности?**
О: После настройки правил Firestore (Часть 4) — да. Твои данные видишь только ты.

**В: Можно установить как приложение на телефон?**
О: Да! Открой в Safari/Chrome → «Поделиться» → «На экран домой». Получится PWA-приложение.

**В: Как сделать бэкап данных?**
О: Firebase Console → Firestore → «Export data». Или напиши скрипт экспорта.

**В: Хочу изменить цвета.**
О: Открой `css/main.css`, раздел `:root {}` в самом начале. Там все цвета. Поменяй значения `--ochre`, `--amber` и т.д.

**В: Как удалить тестовые данные?**
О: Firebase Console → Firestore → найди свои документы → удали вручную. Или в настройках Firestore «Delete collection».
